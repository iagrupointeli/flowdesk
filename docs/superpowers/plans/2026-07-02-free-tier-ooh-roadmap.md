# Free-Tier OOH Roadmap — InteliONE

**Date:** planned 2026-07-01, to be executed from 2026-07-02
**Status:** VISION / not built
**Core constraint:** ZERO paid APIs at runtime. All intelligence is compiled into
deterministic code at dev-time (Claude Code as engineer/architect). The server
runs free, fast, auditable code — no per-token costs, ever.

**Architecture thesis:** intelligence at dev-time, not runtime. Surviving
concepts from the original agent-native design: spatiotemporal inventory
ledger, digital twin of physical assets, commercial graph, job queue — all
deterministic. What dies: LLM as a runtime "employee".

## Phase 2 — Anti-fraud checking (deterministic, no vision LLM)

Classic OOH checking fraud: installer sends an old photo or another point's
photo. Catch it without any LLM:

- **EXIF extraction** (`exif-reader`, free): GPS, timestamp, device.
  - Photo GPS > threshold distance (haversine) from asset geo → flag.
  - Photo timestamp outside campaign window → flag.
- **Perceptual hash** (pHash via `sharp`): same image reused across different
  checkings → automatic flag.
- Output: rule-based compliance score per evidence photo — auditable,
  explainable, zero cost per photo. Feeds the digital twin.

## Track B — dev-time asymmetric features

Criterion: the bottleneck is rare specialist knowledge (optimization,
statistics, OR), not coding effort. Intelligence compiled at dev-time,
deterministic at runtime. All free.

### B1. Commercial package optimizer (set cover / coverage maximization)
Given budget + coverage goal ("R$80k, max reach in Joinville"), pick the
optimal point set. Greedy heuristic with overlap penalty (nearby points
cannibalize each other's audience radius). Depends on Phases 1+4 (geo +
audience score). Runs in milliseconds. This is what agencies pay foreign
OOH-planning tools for.

### B2. Installation routing for MovePro (VRP)
Campaign closes with N points → optimal install order per team/day to
minimize mileage. **Google OR-Tools (open source, runs locally).** Fuel and
per-diem savings are real, measurable money every month — makes InteliONE
read as a profit center to leadership.

### B3. Pricing curve derived from own ledger data
Once the ledger accumulates history (occupancy × closed price × bi-week ×
city): seasonality and elasticity analysis done at dev-time, compiled into
auditable SQL rules (e.g. "BS 40–44 in Balneário historically closes at 92%
occupancy → suggest full price"). No runtime ML.

### B4. National reconciliation as a parser compiler
Per new state: source (dump/spreadsheet/PDF) → I read the mess, write that
state's deterministic parser + tests, it lives in the repo forever. Runtime
never sees an LLM; the repo accumulates one parser per source. In 12 months:
the cleanest OOH dataset in Brazil at ~zero marginal cost.

### B5. Enterprise-grade test suite + hardening
FlowDesk has no automated tests today. Full suite: unit + integration + load
(synthetic 200k-point dataset simulating national scale). Should run in
parallel with everything else, starting now.

**Priority read:** B1 and B2 are the only ones with ROI leadership can *see*
(better proposals, saved fuel) — slot right after Phase 3. B5 runs in
parallel from day one.

## Track C — OOH-specific product features (round 2)

### C1. Category-conflict engine (competitive exclusivity)
Sacred OOH rule no current system validates: no Coke next to Pepsi in the
same bi-week; some contracts demand category exclusivity within a radius.
At booking time: advertiser category × radius × period → block the conflict
before it becomes a client crisis. Deterministic; depends on Phase 1 geo.

### C2. Public landing page per point (organic lead-gen)
The Scoutdoor model proves it works: auto-generate a public page per point
(photo, specs, city, availability indicator, interest form → FlowDesk
intake). ~15k Google-indexable pages for free. Connects directly to the
"GEO — AI ranking for our sites" task in the TI backlog: structured pages
are exactly what AI engines cite.

### C3. Franchise health score
Per-department dashboard: occupancy, checking compliance rate, demand
response time, license status → ranking of the 40+ brands. All derivable
from existing data. Turns InteliONE from ops tool into the holding's
governance instrument (executive meeting material).

### C4. Offline-first PWA for MovePro installers
Real field pain: installer on a highway with no signal cannot upload
checking. Service worker + local queue: photo taken offline, auto-syncs
when back online (EXIF preserved for Phase 2 anti-fraud). Zero infra.

### C5. Photo quality gate (classic CV, no LLM)
Phase 2 complement: blur detection (Laplacian variance), too-dark check,
minimum resolution — reject at upload with a clear "retake now" message to
the installer. Kills the unusable photo discovered only when the advertiser
complains.

### C6. Contract + PI (Pedido de Inserção) generation
Approved campaign → contract and PI PDFs generated from ledger data
(points, bi-weeks, values, terms), numbered and archived in MinIO. Legally
standardizes 40 brands that today each have their own Word file.

## Track D — revenue & strategy features (round 3)

### D1. "Money on the table" report (idle inventory in R$)
`getIdleAssets` already exists but a list of empty points hurts nobody.
Convert: idle points × average market price × vacant bi-weeks =
"R$ 340k left on the table this month, 60% in 4 cities". Same query,
different framing — becomes the metric that opens the sales meeting.

### D2. Renewal radar (churn prevention on bi-week rhythm)
Campaign ending within next 2 bi-weeks with no registered renewal → auto
demand for that brand's salesperson with context (client history, occupied
points, value). In OOH renewal is the profit — sale cost already paid.

### D3. Whitespace map for holding expansion
IBGE (population, income, municipal GDP — free API) × current inventory
density: "cities >100k inhabitants where the group has zero points",
ranked by potential. The map of where to open the next franchise.

### D4. Deterioration detection via pHash drift
Compare perceptual hash of the SAME point's photos across months. Drift
above threshold = something physically changed (torn canvas, dirt,
graffiti) → auto `service_order` (migration 046 tables are waiting for
exactly this). The Estruturas→MovePro predictive loop with zero vision
models — just image math.

### D5. Rate card with discount authority chain
Versioned price table per brand + authority rule: ≤10% seller approves,
above → dept_admin, above 25% → super_admin, with audit trail. Margin
governance — the silent leak of any decentralized 40-brand operation.

### D6. Single client via CNPJ (BrasilAPI, free)
Same national advertiser registered 5 different ways across 5 franchises.
CNPJ normalization + auto-enrichment via BrasilAPI (razão social, CNAE,
address). Without this the commercial graph is born dirty.

### D7. War room TV mode (TvMode.jsx already exists)
Upgrade: live occupancy map + incoming checking feed + franchise scores on
a screen at Blumenau Controllership. Near-zero cost (composes data other
phases generate). Political value: Intelligent Centralization visible on
the wall.

**Priority read:** D1, D2, D5 = highest ROI per effort (direct money).
D3 = most strategic. D7 = cheap political capital for the whole project.

## Track E — Hardening (architecture audit findings, 2026-07-01)

Audit of index.js, database.js, migration runner, auth service/store + prod
deploy observations. Solid foundation (EXCLUDE constraints, transactional
migrations, httpOnly refresh cookie, helmet/CORS, requestId logging, SSRF
guard, streaming uploads w/ magic bytes). Findings below are fixable without
rewrites.

### E-A. Production-critical (fix first)
- ✅ **A1. No rate limiting on /api/auth/login** — DONE 2026-07-02,
  deployed to prod. IP limiter (express-rate-limit, 10/15min) + account
  lockout (migration 048: failed_login_attempts/locked_until, 5 attempts →
  15min, checked before bcrypt.compare) + `app.set('trust proxy', 1)` in
  index.js. Verified externally against prod: 429 triggers exactly on the
  10th cumulative attempt per IP (confirms trust proxy is counting real
  client IP, not the Nginx internal one).
- ✅ **A2. SPA deep links broken in prod** — DONE 2026-07-02, deployed to
  prod. `frontend/public/.htaccess` (mod_rewrite fallback to index.html).
  Verified externally: `GET /admin/map` → 200 (was 404 before).
- ✅ **A3. Uploads likely broken >1MB** — DONE 2026-07-02 (host-side):
  `client_max_body_size 25m;` applied to the one.grupointeli.com nginx
  include.
- **A4. Stateless refresh JWT, no revocation** — logout doesn't invalidate;
  stolen token valid 7d; password change doesn't kill sessions. Fix:
  refresh_tokens table with rotation + reuse detection. **Next up.**
- ✅ **A5. /health unreachable externally** — DONE 2026-07-02 (host-side):
  nginx location for /health added. Verified externally: 200. External
  uptime monitor (UptimeRobot) still not set up.
- ✅ **A6. Backups** — see Phase 0.
- ✅ **A7. socket.io nginx include** — DONE 2026-07-02 (host-side),
  confirmed applied. Verified externally: handshake → 200.

### E-B. Architectural debt (bites at national scale)
- **B1. Background jobs inside the web process** — 5 setIntervals at boot +
  Scoutdoor scraper with in-memory state. Restart kills jobs silently; 2
  instances would double-run everything. Fix: separate PM2 worker +
  pg_try_advisory_lock + job_runs table (aligns with Phase 5).
- **B2. No graceful shutdown** — no SIGTERM handler; every deploy drops
  in-flight requests/uploads. ~15 lines fix.
- **B3. Config duplicated** (.env × ecosystem.config.cjs) — already caused
  SMTP_FROM confusion during deploy. Fix: PM2 uses `node --env-file`;
  single source of truth.
- **B4. Pagination won't survive national scale** — PAGE_SIZE 500 +
  COUNT(*) OVER() + LEFT JOIN demands GROUP BY per search. Fine at 15k,
  degrades at 170k. Prerequisite for national expansion: default 50 +
  keyset pagination.
- **B5. Non-reproducible infra** — minio:latest unpinned; postgres alpine
  lacks pt_BR locale (accent ordering wrong — warning seen in prod logs).
  Pin versions; ICU collation when convenient.
- **B6. demands.service.js god module (~1,200 lines)** — refactor
  opportunistically.

### E-C. Workflow
- **C1. Prod migrations do NOT need root** — PgBouncer gotcha is local-only;
  in prod dev_inteli reaches 127.0.0.1:5432 directly, `npm run migrate`
  works (proven during deploy).
- **C2. No CI** — GitHub Actions free tier: lint + tests (B5) + build per
  push.
- **C3. No log rotation** — pm2-logrotate, 1 command.

**Attack order:** A1+A2 (~1h, closes both public-facing holes) → A5+A3+A7
(single message to host) → A4 → B1–B3 at normal pace.
